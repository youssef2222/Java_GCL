Cahier de charge — Gestion des Congés (CHU Ibn Sina)

Contexte et objectifs

  • Contexte: Application desktop de gestion des congés pour le CHU Ibn Sina.
  • Objectifs:
    • Gérer les employés et leurs droits annuels.
    • Enregistrer les demandes de congé avec calcul des jours ouvrés.
    • Valider/rejeter les demandes, suivre le solde annuel.
    • Visualiser les demandes en attente et approuvées.
    • Branding avec logo et lancement via icône bureau.

Périmètre fonctionnel

  • Gestion des employés: créer/afficher des employés avec id, nom, droit annuel; écran Employés: ajout + liste.
  • Demandes de congé: saisie ID employé, date début, date fin, type; calcul des jours ouvrés; numéro lisible auto-incrémenté.
  • Validation: liste En Attente (Approuver/Rejeter avec motif), liste Approuvés avec N°, dates, type, jours.
  • Solde: affichage du solde annuel restant par employé et année; diminution par congés ANNUAL approuvés.
  • Branding et lancement: logo dans l’en-tête et icône de fenêtre; icône bureau Windows (lancement silencieux).

Règles métier

  • Jours ouvrés: compter tous les jours de start à end inclus; exclure samedi et dimanche.
  • Types: ANNUAL, SICK, UNPAID, MATERNITY, PATERNITY.
  • Statuts: PENDING, APPROVED, REJECTED.
  • Solde annuel: droit_annuel − somme(jours des demandes ANNUAL APPROVED sur l’année), borné à 0.
  • Identifiants: id technique (UUID), display_id (entier auto-incrémenté affiché).

Modèle de données (SQLite)

  • employees: id (PK), name, annual_entitlement.
  • leave_requests: id (UNIQUE), display_id, employee_id (FK→employees.id), start_date, end_date, type, year, days, status, rejection_reason.
  • sequences: name (PK), value.

Architecture technique

  • Java SE + Swing (desktop), SQLite via JDBC, pattern Service/Repository.
  • Entrée: Main (initialisation DB, modes gui/demo).
  • Service: demande, validation, solde, listes En Attente/Approuvés.
  • DB: schéma et séquences.
  • UI: onglets Employés, Demande, Validation, Solde; logo loader.

Interfaces utilisateur

  • Employés: formulaire d’ajout; tableau non modifiable; rafraîchir.
  • Demande Congé: saisie; message de succès avec N°, Jours, Type.
  • Validation: En Attente (actions Approuver/Rejeter), Approuvés (N°, dates, type, jours).
  • Solde Annuel: saisie ID employé + année; affichage du solde.

Exigences non fonctionnelles

  • Performance: réactivité locale, requêtes SQLite rapides.
  • Fiabilité: persistance des statuts; IDs lisibles persistés.
  • Sécurité: pas de secrets en clair.
  • Ergonomie: tableaux non éditables; formulaires clairs; IDs lisibles.
  • Maintenance: architecture modulaire; interfaces repository.

Déploiement et lancement

  • Prérequis: Java; SQLite JDBC `lib/sqlite-jdbc-3.51.1.0.jar`.
  • Compilation/Exécution:
    • javac -cp lib/sqlite-jdbc-3.51.1.0.jar -d out (Get-ChildItem src/main/java -Recurse -Filter *.java).FullName
    • java -cp "out;lib/sqlite-jdbc-3.51.1.0.jar" com.gcl.conge.Main gui
  • Icône bureau: run_gui.bat + run_gui.vbs, raccourci Windows, lancement silencieux.

Tests et acceptation

  • Ajout employé et affichage.
  • Demande ANNUAL sur une semaine: jours=5 (lun–ven).
  • Validation et solde annuel.
  • Listes En Attente/Approuvés cohérentes.
  • Critères: calcul correct, solde juste, ID lisible, lancement silencieux.
